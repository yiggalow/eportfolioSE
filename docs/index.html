What is Jira ?

A bug tracing and project management software that ...

  1. Manage Agile and Scrum teams
  2. Organize your project tasks
  3. Capture and record software bugs

History of Jira

  1. Launched by Atlassian in 2002
  2. Named by Gojira/Godzilla
  3. firstly only a issue tracking software for developer
  4. later adopted by a non-IT organization as a pm tool
  5. 2012 third party dev. plugins

Which packages do they offer ?

  1. Jira Core: The basic Jira project and task management platform for non-technical users
  2. Jira Software: A specialized version of the tool, specially built for software developers
  3. Jira Service Desk: A platform for customer support and IT professionals. While Service Desk is a key JIRA package, it’s not a project management tool

Who can use it ?

Jira can be used by all types of teams and businesses

  1. Software Development: Developers use Jira’s features to manage their projects using Scrum or Kanban boards
  2. Marketing: Marketers use Jira to manage complex projects like events and product launches (which involve working with multiple teams)
  3. HR: HR professionals can streamline their hiring and onboarding processes by creating custom workflows with Jira

The Benefits of Jira

Built for Agile and Scrum Management

  1. User stories: Each software product feature that needs to be developed
  2. Story Points: The effort you’ll need to complete your user stories
  3. Sprints: The time taken to complete a section of the project. A sprint can last anywhere from 1 – 4 weeks
  4. Easily handling because of Drag and Drop
